02 — Command Reference

cmd desktop mobile notes
bal bal [--acct] bal mobile drops account flag, defaults to checking
send send <amt> to <user> [--memo] [--recurring] guided chips: amount → to → confirm flags become structured form fields; confirm step always shown on mobile
tx tx [--last n] | grep <term> tx + tap-to-filter pills piping is desktop-only — too fiddly to type on glass
statement statement --month=may --format=pdf statement → month picker sheet mobile swaps flags for a native date picker
freeze freeze card freeze (single tap, no args) security actions simplified to zero-argument on mobile by design
help help [<cmd>] persistent chip tray (always visible) mobile solves discoverability structurally — chip row replaces man pages

03 — Research Brief

hypothesis

A constrained, text-based financial interface increases perceived transparency and control at the cost of first-use discoverability.

The desktop/mobile split is justified if power-user trust rises with repeated use — not by removing friction entirely, but by making it progressive.

method

A/B moderated usability test — terminal prototype vs. standard GUI banking mockup.

Same three tasks: check balance, send money, find a transaction. Think-aloud protocol throughout.

Two sessions per participant: day 0 (first use), day 7 (return use).

metrics

Task completion time per condition — captures discoverability cost.

SUS score per condition.

Trust & control self-report scale pre/post — this is the differentiating metric most usability studies skip.

Error rate on first vs. return session.
